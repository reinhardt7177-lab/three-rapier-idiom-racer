# 무무 시티 익스프레스 — 레이싱 손맛(트랙 × 속도 시스템) 계획

배달 게임이어도 심장은 레이싱이다. 이 문서는 "밟는 맛 · 꺾는 맛 · 스치는 맛"
세 가지 감각을 살리기 위한 진단과 단계별 구현 계획이다.
비주얼 계획(visual-quality-plan.md)과 같은 **만들기 → 계측 → 통과 → 커밋** 루프를 쓴다.

## 1. 현재 시스템 실측 진단

코드 기준 (`create-delivery-runtime.js`):

| 항목 | 현재 값 | 판정 |
|------|---------|------|
| 속도 단위 | 1 world u/s = 5 km/h, 기본차 최고 40 u/s(200km/h), 포르쉐 60 u/s | 유지 |
| 가속 | 기본 8.8 u/s² + 보너스, 최고속 근처 22%까지 감쇠 | 곡선은 좋으나 "기어 변속" 단이 없어 밋밋 |
| **터보** | +14.5 u/s²지만 **최고속도에 클램프** → 최고속에서 터보를 눌러도 아무 일도 없음 | **최우선 수술 대상** |
| 터보 게이지 | 소모 30/s, 회복 10/s | 리스크·보상 구조 없음 (아무 때나 눌러도 됨) |
| 조향 | yawRate 모델, 속도 오를수록 grip 감소(1.08→0.46) | 안정적이지만 미끄러지는 재미(드리프트) 전무 |
| 카메라 | FOV 50 + speedRatio×20, 정지 시 47 | 방향은 맞으나 터보 킥·셰이크 없음 |
| 사운드 | 원샷 효과음뿐, **엔진 루프 없음** | 속도의 절반은 소리다 — 신규 제작 필요 |
| AI 교통 | 장식(충돌·니어미스 판정 없음) | 스치는 맛 재료가 놀고 있음 |
| 트랙 안내 | 차량 위 내비 화살표뿐 | 코너 예고·감속 지점이 없어 도로가 "트랙"으로 안 읽힘 |

## 2. 목표 감각 정의

1. **밟는 맛** — 출발은 묵직하게 붙고, 기어가 올라갈 때마다 한 번씩 "울컥"한다.
   터보는 게이지를 태우는 대신 최고속의 벽을 뚫는다.
2. **꺾는 맛** — 고속 급조향이면 뒤가 흐르며 드리프트로 전환, 카운터로 잡으면
   보너스가 붙는다. 실패해도 스핀 아웃이 아니라 감속 페널티(아케이드 관용).
3. **스치는 맛** — 교통 차량 옆을 고속으로 스치면 NEAR MISS 보너스,
   정면으로 박으면 감속 + 셰이크. 도로가 긴장의 원천이 된다.

## 3. 단계별 구현 계획

### R-A. 터보 오버드라이브 (즉효 최대 — 최우선)

- 터보 중 속도 상한을 `maxForward × 1.15`로 확장, 터보 해제 시 2.2/s로 자연 감속
- 터보 시작 순간: FOV +9 펀치(0.25초 뒤 속도 FOV로 수렴), 화면 미세 셰이크(진폭 0.05, 6Hz 감쇠)
- 터보 게이지에 리스크 부여: 잔량 25% 미만에서 사용 시 회복 지연 2초(과열)
- HUD 속도계: 오버스피드 구간에서 숫자·게이지 주황→빨강 전환

**계측**: Playwright로 HUD km/h를 100ms 간격 샘플링 →
터보 ON에서 최고속 대비 +15%±1 도달, OFF 후 3초 내 원래 상한 복귀.
**통과 기준**: 최고속 주행 중 터보를 눌렀을 때 체감 변화(속도·FOV·색)가 3중으로 발생.

### R-B. 엔진 사운드 루프 + 가상 기어

- WebAudio 상시 루프: 톱니파 2개(디튠 5센트) + 로우패스, 피치 = 기어 내 RPM 비율
- 가상 4단 기어: 속도 구간을 4분할, 기어 상승 시 RPM 65%로 리셋 + 가속 0.12초 미세 딥
  → 가속 곡선에 "단"이 생기고 소리가 계단식으로 차오름
- 터보 중: 피치 +12%, 화이트노이즈 흡기음 레이어
- 뮤트 설정은 기존 setMuted에 연동

**계측**: 콘솔 에러 0건, 기어 시프트 시점 로그가 속도 구간과 일치.
**통과 기준**: 눈 감고 소리만으로 가속·터보·감속을 구분할 수 있다.

### R-C. 드리프트 시스템 (꺾는 맛)

- 진입 조건: `speedRatio > 0.55` 그리고 조향 유지 0.25초 이상 + 브레이크 탭(또는 전용 키)
- 드리프트 중: 차체 yaw가 진행 방향보다 최대 18° 더 돌아감(시각적 슬립각),
  실제 진행 방향은 완만히 회전 — 관성 느낌은 내되 조작 난도는 아케이드로
- 뒷바퀴 위치에 타이어 스모크 파티클(InstancedMesh 12개 재활용) + 노면 스키드 마크 2줄
- 드리프트 유지 시간에 비례해 콤보 점수(0.5초당 +40), 성공 종료 시 터보 게이지 +8
- 실패(도로 이탈)는 기존 소프트 복귀를 그대로 사용 — 벌은 가볍게, 보상은 확실하게

**계측**: 교차로 진입 시나리오 자동 주행으로 드리프트 진입/종료 로그 확인, 스크린샷에 스모크·스키드 노출.
**통과 기준**: 90° 코너를 감속 없이 드리프트로 통과하는 라인이 성립하고, 점수 보상이 HUD에 뜬다.

### R-D. 니어미스 & 교통 상호작용 (스치는 맛)

- 교통 차량과 거리 판정 추가: 측면 1.6m 이내 + `speedRatio > 0.5` 통과 시
  "NEAR MISS +60" 팝업 + 터보 게이지 +5 (0.8초 쿨다운)
- 정면 겹침(1.1m 이내)은 충돌: 속도 45%로 감쇠 + 셰이크 + bump 사운드 강화판
- 니어미스 연속 성공은 콤보 배율(×1, ×1.5, ×2 — 8초 유지)

**계측**: 교통 밀집 구간 자동 주행에서 니어미스/충돌 이벤트 로그와 점수 변화 대조.
**통과 기준**: 교통을 "피해 다니는 장애물"이 아니라 "점수 재료"로 쓰고 싶어진다.

### R-E. 트랙 리더빌리티 (도로를 트랙으로)

- 곡률 분석: 도로 폴리라인에서 회전각 상위 코너를 뽑아 **코너 예고 표지판**(쉐브론 ‹‹‹)을
  진입 30m 앞 인도 옆에 자동 배치 (InstancedMesh)
- 급코너 바깥쪽 연석에 **적백 럼블 스트립** 텍스처 밴드
- 배송 경로 위 다음 코너에 한해 노면에 반투명 **레이싱 라인 곡선**(기존 내비 색과 통일)
- 직선 장거리 구간(아우터 벨트)에 거리표지판·가로등 등주기 소품 → 고속 파랄락스 강화

**계측**: check-data에 "코너 표지 침범 검사"(도로면·건물과 겹침 0건) 추가, 교차로·코너 스크린샷.
**통과 기준**: 처음 달리는 코너에서도 표지만 보고 감속 타이밍을 잡을 수 있다.

### R-F. 속도 연출 마감 (카메라·파티클)

- 카메라: speedRatio 0.75 이상에서 미세 셰이크(진폭 0.03·랜덤 시드), 터보 시 후방 거리 +1.2m
- 화면 가장자리 스피드 라인 파티클(터보·오버스피드 전용, 세로 화면에서는 밀도 50%)
- 뒷범퍼 배기 파티클: 가속 시 2발, 터보 시 화염 스프라이트
- 기존 FOV 곡선 유지하되 상한 70→74로 소폭 확장 (오버스피드 구간 전용)

**계측**: S3~S6 스크린샷 세트 + 세로 화면에서 HUD 가림 없음 확인.
**통과 기준**: 같은 200km/h라도 터보 중인지 아닌지 스크린샷만으로 구분된다.

### R-G. 텔레메트리 회귀 하니스 (전 단계의 안전망)

- `scripts/telemetry.mjs`: Playwright 자동 주행 30초 동안 km/h·기어·드리프트·니어미스
  이벤트를 JSON으로 덤프
- 기준선 검증: 기본차 0→100km/h 2.5~3.5초 / 0→200km/h 9~12초 / 터보 최고속 +15%±1 /
  드리프트 중 평균 감속 15% 이하
- 수치 이탈 시 실패 종료 → 튜닝 커밋마다 실행해 "손맛 회귀"를 숫자로 방어

## 4. 진행 순서와 원칙

```
R-A 터보 → R-G 하니스 → R-B 사운드 → R-C 드리프트 → R-D 니어미스 → R-E 트랙 → R-F 연출
```

- R-A가 가장 싸고 체감이 크므로 첫 커밋. R-G를 바로 뒤에 세워 이후 튜닝을 계측으로 보호.
- 물리 모델은 교체하지 않는다 — 지금의 yawRate 아케이드 모델 위에 겹만 얹는다
  (Rapier 도입은 손맛 완성 후 별도 결정).
- 모든 수치는 이 문서의 표를 갱신하며 튜닝한다. 코드에 매직 넘버를 흩뿌리지 않고
  `DRIVE_TUNING` 상수 오브젝트 하나로 모은다.
- 한 단계 = 한 커밋 = 텔레메트리 + 스크린샷 통과.
