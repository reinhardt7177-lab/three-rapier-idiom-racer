# 무무 시티 익스프레스 — 비주얼 품질 개선 계획

City Racer풍 로우폴리 아트로 "제대로 나올 때까지" 반복하기 위한 제작·점검 계획.
모든 단계는 **만들기 → 스크린샷 점검 → 기준 통과 → 커밋** 순서로 진행한다.

## 0. 반복 루프 (모든 단계 공통)

한 단계를 끝낼 때마다 아래 6장 스크린샷 세트를 같은 조건으로 다시 찍어 비교한다.

| # | 장면 | 잡는 방법 |
|---|------|-----------|
| S1 | 차고(타이틀) 가로 1280×720 | 첫 로딩 후 4초 대기 |
| S2 | 차고 세로 390×844 | 모바일 뷰포트 |
| S3 | 출발 직후 3인칭 후방 | 배송 출발 → 2초 |
| S4 | 주행 중 교차로 진입 | 가속 6초 후 |
| S5 | 차량 옆면 클로즈업 | 마우스 드래그로 카메라 90° 회전 |
| S6 | 야간(스카이라인 미션) | 미션 3 선택 후 주행 |

자동 점검(스크린샷 전에 필수):

```bash
node scripts/check-data.mjs   # 데이터·도로 무결성
npm run build                 # 번들 통과
# Playwright 주행 스크립트에서 콘솔 에러 0건 확인
```

통과 기준(공통): 콘솔 에러 0건, 세로/가로 모두 HUD가 3D 요소를 가리지 않음,
프레임이 눈에 띄게 끊기지 않음(도시 전경에서 InstancedMesh 유지).

## 1. 완료된 작업 (2026-07-18)

- Meshy GLB 차량 제거 → 절차 로우폴리 차량 단일화, 바퀴 펜더 안 수납 + 휠 아치
- AI 교통 차량 바퀴 트랙 축소 (±1.08 → ±0.92)
- 건물 창문 그리드(전·후·측면), 1층 쇼윈도, 옥상 설비, 파스텔 팔레트
- 도로 양측 연석 인도(아스팔트보다 +18cm), 3거리 이상 교차로 횡단보도
- GLB 40MB·Meshy 스크립트 삭제, check-data의 GLB 검사를 차량 데이터 검사로 교체

## 2. 남은 문제 (스크린샷 근거)

| 문제 | 근거 장면 | 원인 |
|------|-----------|------|
| P1. 기본 차량이 거의 검게 보임 | S3·S5 | 메탈 그레이(#5a6573) + 낮은 주변광에서 명도 부족, 유리·다크 트림 비중 과다 |
| P2. 캐빈 유리가 불투명 검정 | S3 | glassMaterial 0x07131e, 투명도 없음 |
| P3. 시내 일부 구역이 청회색 일색 | S4 | 스카이라인(원경) 타워 팔레트가 구식 그대로 |
| P4. 나무가 단일 원뿔이라 밋밋 | S4 | ConeGeometry 1단 |
| P5. 교차로 원판이 도로와 이질적 | S4 | 실린더 36각 + 정지선 없음 |
| P6. 코인이 도넛(토러스) 모양 | S3 | TorusGeometry — 동전은 납작 실린더가 맞음 |
| P7. 야간 장면 미점검 | S6 없음 | 미션 3 야간 주행 스크린샷을 아직 안 찍음 |

## 3. 단계별 계획

### 단계 A — 차량 가독성 (P1·P2)

작업:
- 기본 페인트 명도 상향(#5a6573 → #8a97a8 수준) 또는 기본 선택을 밝은 색으로 변경
- 차체 emissiveIntensity 0.055 → 0.09로 올려 그늘에서도 실루엣 유지
- glassMaterial에 `transparent: true, opacity: 0.86` + 밝은 틴트(0x1a3346)
- 다크 트림(스커트·디퓨저) 명도 소폭 상향(0x111820 → 0x1c2733)

점검: S3·S5에서 차체·유리·트림 3톤이 구분되는지, 5색 페인트 전부 차고에서 확인.
통과 기준: 정오 조명에서 차체가 검정으로 뭉개지지 않고, 유리에 하늘 반사 톤이 보임.

### 단계 B — 원경 스카이라인 팔레트 통일 (P3)

작업: createSkyline의 skylineColors를 시내 지구 파스텔 팔레트와 같은 계열로 교체,
크라운 발광색은 유지(야간 스카이라인 포인트).

점검: S4에서 근경 건물과 원경 타워의 톤이 이어지는지.
통과 기준: 화면 안에 "회청색 벽" 느낌의 연속 구간이 없음.

### 단계 C — 거리 소품 밀도 (P4·P5·P6)

작업:
- 나무를 2단 원뿔 + 원통 줄기로 교체(InstancedMesh 유지, 인스턴스 수 동일)
- 교차로 진입부에 정지선(횡단보도 앞 흰 굵은 선) 추가
- 코인을 납작한 CylinderGeometry(r 0.62, h 0.18)로 교체, 회전 애니메이션 유지
- 가로등·벤치는 인도 위 일정 간격 배치(선택 — 프레임 여유 확인 후)

점검: S4 교차로 장면에서 정지선·횡단보도·인도가 한 세트로 읽히는지.
통과 기준: 교차로에서 "어디서 멈추는지"가 그림만으로 읽힘. 프레임 저하 없음.

### 단계 D — 야간 품질 (P7)

작업 없이 먼저 점검: 미션 3(스카이라인 에너지 작전)으로 S6 촬영.
문제 발견 시: 창문 발광 강도, 가로등 유무, 차 헤드라이트 노면 반사 순으로 보강.

통과 기준: 야간에도 도로 경계·차선·목적지 마커가 또렷하고 차량 실루엣이 보임.

### 단계 E — 최종 회귀 점검·머지

- S1–S6 전체 세트 재촬영 후 이전 세트와 나란히 비교
- check-data + build + 콘솔 에러 0건 재확인
- 사용자 확인 후 main 머지 → Vercel 반영

## 4. 작업 원칙

- 한 단계 = 한 커밋. 스크린샷 통과 전에는 다음 단계로 넘어가지 않는다.
- 지오메트리 추가는 InstancedMesh 우선, 개별 Mesh 남발 금지(프레임 예산).
- 색은 지구(district) 팔레트에서만 뽑아 도시 전체 톤을 유지한다.
- check-data.mjs가 지키는 도로·건물 침범 규칙을 시각 변경이 깨지 않게 한다.
