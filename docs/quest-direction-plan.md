# 무무 시티 익스프레스 — 퀘스트 방향성 재점검과 새 설계

레이싱 손맛(racing-feel-plan)과 비주얼(visual-quality-plan)은 자리를 잡았다.
이제 병목은 **콘텐츠 구조**다. 이 문서는 현재 퀘스트 루프의 문제를 코드 기준으로
진단하고, 새 방향 "달리면서 배우는 커리어"를 제안한다.

## 1. 현재 구조 진단 (코드 실측)

현재 루프: 미션 선택(3개 고정) → 배송지 3곳 순회 → 도착마다 정지 후 모달 퀴즈 1문
→ 완주 시 골드 → 차량 5대·튜닝 구매.

| # | 문제 | 근거 |
|---|------|------|
| P1 | **콘텐츠 10분 소진** | 미션 3개·배송지 5곳·경로 고정. 두세 판이면 전부 경험, 재도전 이유 없음 |
| P2 | **이코노미 단선** | 골드 지급은 미션 완주 단 1곳. 퀴즈 정답(+600)·니어미스·드리프트·코인은 전부 "점수"인데 점수는 쓸 곳이 없음. 차 5대 사면 경제 종료 |
| P3 | **퀴즈가 흐름을 끊음** | 도착 → 강제 정지 → 모달 4지선다 → 재출발. 애써 만든 속도감·드리프트와 완전히 단절 |
| P4 | **긴장 부재** | 실패 조건이 시간초과뿐. 경쟁자·라이벌 없음. 타임 보너스(+10초)로 사실상 실패 없음 |
| P5 | **학습 루프 없음** | 사자성어 100개 중 매번 랜덤 1문. 틀린 문제 재출제 없음 → "학습"이 아니라 통행세 |

## 2. 새 방향: "달리면서 배우는 커리어" (5 필러)

컨셉 한 줄 — **배달은 계약이고, 사자성어는 도로 위에 있고, 승급전엔 라이벌이 있다.**

### 필러 1. 절차 생성 계약 (P1 해결)
고정 미션 3개를 "계약 보드"로 교체. 매 판 3장의 계약이 절차 생성된다:
- 배송지 조합·순서 무작위 (기존 5곳 + 경로 그래프 재사용)
- 제한시간 = 경로 실거리 ÷ 목표 평균속도 (텔레메트리 기준으로 산출)
- 보너스 조건 1개 부착: 무충돌 / 니어미스 5회 / 드리프트 누적 3초 / 코인 10개
- 등급이 오르면 배송지 수(2→4)·목표 속도·보상이 함께 상승

### 필러 2. 도로 위 사자성어 게이트 (P3·P5 해결의 절반)
모달 퀴즈를 **주행 중 게이트 선택**으로 이동:
- 배송 경로 중간에 2~3개의 아치 게이트 지점 생성. 머리 위 표지판에 문제
  ("苦盡甘來의 뜻은?"), 각 차선 게이트에 보기 1개씩
- 정답 차선 통과: 터보 +25, 시간 +6초, 골드 +30 — **속도를 잃지 않는 보상**
- 오답 차선: 3초 저속 페널티(스틸 없음, 아케이드 관용)
- 배송 도착 모달 퀴즈는 "마지막 배송지 1회"만 남겨 보스 문제화

### 필러 3. 오답 수배 전단 (P5 해결)
틀린 사자성어는 로컬 저장 "수배 목록"에 들어가고, 다음 계약의 게이트에
우선 재출제된다(간격 반복). 맞히면 수배 해제 + 보상 1.5배.
→ 게임을 오래 할수록 자기가 약한 사자성어만 골라서 다시 만나게 된다.

### 필러 4. 평판 등급과 라이벌 승급전 (P4 해결)
- 점수를 **평판 XP**로 전환(쓸 곳이 생김): 루키 → 브론즈 → 실버 → 골드 → 레전드
- 등급 승급 조건 도달 시 **승급전**: 라이벌 AI가 같은 배송 계약을 동시에 뛴다
  (경로 그래프를 따라 달리는 고스트, 등급별 목표 평균속도로 주행)
- 먼저 완주하면 승급 + 라이벌 차량 도장 해금. 지면 재도전(계약은 새로 생성)
- 차량 해금을 가격+등급 조건으로: 포르쉐는 골드 등급부터 구매 가능

### 필러 5. 이코노미 리워크 (P2 해결)
| 재화 | 얻는 곳 | 쓰는 곳 |
|------|---------|---------|
| 골드 | 계약 완주, 보너스 조건, 게이트 정답, 수배 해제 | 차량 구매, 튜닝(레벨 상한 5→7), 도장·스티커 일부 유료화 |
| 평판 XP | 점수 전량 전환(니어미스·드리프트·코인·시간 보너스) | 등급 승급 → 계약 티어·차량 해금 |
| 별 토큰 | 도로 코인 | 게이트에서 힌트 1개 소거(보기 4→3) — 학습 보조로 순환 |

## 3. 구현 계획 (Q 단계)

| 단계 | 내용 | 핵심 파일 | 통과 기준 |
|------|------|-----------|-----------|
| Q-A 계약 생성기 | 시드 기반 계약 3장 생성(배송지 조합·시간·보너스 조건), 계약 보드 UI로 미션 카드 교체 | game-data.js(generator), App.jsx | check-data에 "생성 계약 100개 전수 검사"(경로 존재·시간 하한) 추가 통과 |
| Q-B 사자성어 게이트 | 경로 중간 게이트 배치·차선 판정·보상/페널티, 도착 퀴즈는 최종 배송지만 | create-delivery-runtime.js | 게이트 통과가 정지 없이 이뤄지고, 정답/오답 피드백이 주행 중 HUD로 보임 |
| Q-C 수배 전단 | 오답 저장(localStorage)·우선 재출제·해제 보상 | App.jsx, runtime | 같은 문제를 틀리면 다음 판 게이트에 반드시 재등장 |
| Q-D 평판·등급 | XP 전환, 등급 UI, 차량 등급 잠금, 저장 마이그레이션 | App.jsx, game-data.js | 기존 세이브(gold/owned)가 깨지지 않고 승계 |
| Q-E 라이벌 승급전 | 경로 추종 고스트 카, 실시간 순위, 승급 판정 | runtime | 승급전에서 라이벌이 경로를 이탈 없이 완주, 승/패 판정 정확 |
| Q-F 밸런스 계측 | telemetry.mjs 확장: 생성 계약 자동 플레이로 클리어율 로그 | scripts | 루키 계약 클리어율이 기본 차량 기준 "널널", 상위 티어는 튜닝 요구 |

순서는 Q-A → Q-B가 최우선(재플레이성과 흐름 회복이 가장 큰 구멍).
Q-C는 Q-B에 얹는 소품. Q-D·Q-E는 큰 덩어리라 별도 라운드로.

## 4. 지키는 것

- 기존 레이싱 손맛(터보·드리프트·니어미스)은 계약 보너스 조건과 게이트 보상으로
  **퀘스트에 편입**한다 — 시스템을 새로 만들지 않고 쓸모를 부여
- 사자성어 데이터(100개)·도로 그래프·배송지 5곳은 그대로 재사용
- check-data가 지키는 무결성 규칙(경로 존재·도로 침범)은 계약 생성기에도 적용
- 모바일 세로 UI에서 게이트 표지판 가독성 확보(폰트 크기 우선 검증)
