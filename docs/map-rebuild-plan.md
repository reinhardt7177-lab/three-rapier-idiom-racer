# 무무 시티 익스프레스 — 맵 리빌딩 방향 계획

퀘스트 리워크(quest-direction-plan)가 "무엇을 하며 달리는가"라면, 이 문서는
"어디를 달리는가"다. 현재 맵을 실측 진단하고, 레이싱 손맛·계약·승급전을
받쳐 줄 도시 구조로 리빌딩하는 계획을 정리한다.

## 1. 현재 맵 실측 진단

- 월드 660×660 (CITY_HALF 330), 도로 그래프: 허브 방사 + 내부 링 + 외곽 벨트
  + 대각 강 1줄(다리 3개) + 지구별 지선. 도로 폭 4종(23/16/13.5/10.5m)
- 지형: 완만한 자연 기복 + 도로 주변 토공 블렌딩. 다리만 아치(+1.8m)

| # | 문제 | 왜 문제인가 |
|---|------|------------|
| M1 | **수직성 부재** | 고가·터널·언덕 스위치백이 없어 어느 길이든 같은 높이의 평면 주행. 200km/h 오버드라이브에서 시야 변화가 없다 |
| M2 | **시그니처 구간 없음** | "그 코너"라고 부를 수 있는 헤어핀·S커브·초장직선이 설계돼 있지 않다. 코너는 있지만 기억에 남지 않는다 |
| M3 | **지름길·도박 루트 없음** | 모든 경로가 내비 최단경로 그대로. 실력으로 시간을 버는 선택지(좁은 골목, 점프 램프)가 없다 |
| M4 | **지구별 주행 성격 동일** | 팔레트만 다르고 코너 리듬·도로폭 구성은 전 지구 동일. "어디를 달리는지"가 몸으로 안 느껴진다 |
| M5 | **승급전 서킷 부재** | 라이벌 승급전(퀘스트 계획 Q-E)에 쓸 순환 노선이 없다. 외곽 벨트가 있지만 단조로운 타원 |

## 2. 리빌딩 방향: "수직으로, 성격 있게, 도박 가능하게"

원칙: **그래프 구조(city-map.js)와 check-data 무결성 규칙은 유지**하고,
노드·도로·지형 함수를 확장한다. 전면 재작성이 아니라 골격 위 증축.

### 축 1. 스카이웨이 (M1·M5 해결)
- 외곽 벨트 중 동·남 구간을 **고가 고속도로**로 승격: 지상 +9m, 진출입 램프 4개
- 구현은 기존 bridge 아치 시스템의 일반화 — `road.elevation` 필드 추가,
  `roadBaseHeightAt`이 램프 구간에서 스플라인 보간
- 고가 아래는 기둥(인스턴스) + 지상 지선 통과 → 도시에 2층이 생긴다
- 스카이웨이 1바퀴 = 승급전 서킷. 직선 40%·스윕 40%·램프 20%로 리듬 설계

### 축 2. 시그니처 구간 3종 (M2 해결)
1. **하버 헤어핀**: 항구 남단에 180° 헤어핀 + 적백 럼블 스트립 + 쉐브론 집중 배치.
   드리프트 보너스 조건 계약이 이 구간을 지나게 설계
2. **리버사이드 S커브**: 강 북안을 따라 S자 3연속(경관 도로 폭 13.5m 재사용).
   니어미스 조건 계약의 무대 — 교통 루프를 이 구간에 추가
3. **센트럴 그랜드 스트레이트**: 허브→북링 직선을 폭 23m 그대로 400m급으로 연장.
   사자성어 게이트(퀘스트 Q-B)의 기본 배치 구간 — 직선이라 읽고 고를 시간이 있다

### 축 3. 지름길·도박 루트 (M3 해결)
- **골목 숏컷 4개**: 새 도로 타입 `alley`(폭 7m, 교통 없음, 커브 급함).
  내비는 안내하지 않지만 미니맵에 점선으로 표시 — 아는 사람만 쓰는 길
- **점프 램프 1개**: 서쪽 다리 상류에 강을 건너뛰는 램프. 성공하면 다리 우회
  대비 -6초, 실속 착수하면 저속 복귀 페널티. 물리는 단순 포물선(y속도 + 중력)
- check-data 확장: alley도 교차 검사에 포함, 램프 착지점이 도로 위인지 전수 검사

### 축 4. 지구별 주행 성격 (M4 해결)
| 지구 | 도로 구성 | 주행 리듬 |
|------|-----------|----------|
| 센트럴 | 간선 직교 + 신호 교차로 밀집 | 스톱&고, 게이트 집중 |
| 하버 프론트 | 넓은 스윕 + 헤어핀 | 고속 진입 → 풀 브레이킹 |
| 웨스트 주택가 | 골목 밀도 최대, 지선 위주 | 테크니컬, 숏컷 보상 |
| 이스트 테크 | 완만한 곡선 캠퍼스로 | 리듬 코너링 |
| 스카이웨이 | 고가 순환 | 오버드라이브 전용 무대 |

### 축 5. 랜드마크 세트피스 (지구 인지 보조)
- 하버: 대형 크레인 2기 확장 + 정박 화물선 1척(로우폴리 박스 조합)
- 센트럴: 스타디움(타원 셸) — 승급전 시상 연출 배경
- 주택가: 급수탑 언덕 + 공원 대관람차(회전 애니메이션, 원거리 시인성)
- 전부 InstancedMesh/저폴리 조합, 도로 침범 검사 대상에 포함

## 3. 구현 단계

| 단계 | 내용 | 통과 기준 |
|------|------|-----------|
| M-A 그래프 증축 | 노드·도로 추가(그랜드 스트레이트, S커브, 헤어핀, alley 4개), 지구별 폭 재배정 | check-data 전수 통과(교차·표면 충돌 0), 기존 5개 배송지 경로 전부 유효 |
| M-B 스카이웨이 | `elevation` 필드 + 램프 보간, 기둥 인스턴스, 고가 전용 텔레메트리 랩 | 스카이웨이 1바퀴 자동 주행 완주, 지상 도로와 표면 충돌 0 |
| M-C 숏컷·램프 | alley 타입, 점프 램프 포물선 물리, 미니맵 점선 | 램프 착지 성공/실속 판정 정확, 숏컷 경유 시 실측 -4초 이상 이득 |
| M-D 세트피스 | 스타디움·크레인·화물선·대관람차 | 스크린샷에서 지구 구분이 실루엣만으로 가능 |
| M-E 미니맵 리제너레이션 | SVG 장식(타원)을 도로 그래프 기반 자동 렌더로 교체 | 새 도로가 미니맵에 자동 반영, alley는 점선 |
| M-F 통합 검증 | 텔레메트리 서킷 랩 + 계약 생성기(Q-A)와 교차 검증 | 전 계약 경로 유효, 성능(인스턴스 수) 리그레션 없음 |

## 4. 퀘스트 계획과의 결합 지점

- 그랜드 스트레이트 = 사자성어 게이트 기본 무대 (Q-B)
- 헤어핀·S커브 = 드리프트/니어미스 보너스 계약의 지정 구간 (Q-A)
- 스카이웨이 = 라이벌 승급전 서킷 (Q-E)
- 숏컷·램프 = 상위 등급 계약의 시간 설계에 반영 — 아는 사람이 이기는 맵

## 5. 순서 제안

퀘스트 Q-A/Q-B(계약·게이트)가 먼저 — 지금 맵으로도 동작한다.
맵은 M-A(그래프 증축)부터 별도 라운드로 시작해 Q와 병렬 진행 가능하되,
M-B 스카이웨이는 Q-E 승급전 직전에 맞추면 된다.
